# -Планировщик задач ОСРВ» (RTOS)-
Этот проект - симулятор планировщика реального времени, реализующий классический алгоритм Rate Monotonic Scheduling (RMS). Его цель - продемонстрировать детерминизм и предсказуемость алгоритма RMS, свойственные системам реального времени. Инструмент не просто моделирует работу планировщика, но и предоставляет все необходимое для его верификации и глубокого изучения.

Название: Планировщик задач ОСРВ» (RTOS)
* Язык: C++17 (STL, стандартные контейнеры, системный WinAPI для работы с кодировкой, файловой системой и таймерами)
* Назначение: Образовательный инструмент, который позволяет сгенерировать набор периодических задач и наглядно пронаблюдать за их выполнением планировщиком RMS. Включает систему аутентификации, логирование, экспорт данных, сохранение/загрузку конфигураций и набор юнит-тестов для проверки ключевых свойств алгоритма.

## Алгоритм Rate Monotonic Scheduling (RMS)

(RMS) - это статический алгоритм планирования с фиксированными приоритетами, где каждая периодическая задача получает приоритет, обратно пропорциональный своему периоду: чем короче период, тем выше приоритет. Основой для анализа планируемости служит теорема Liu & Layland, согласно которой набор задач гарантированно планируем, если загрузка процессора не превышает определенный порог.

В дипломной работе оказано, что разработанный планировщик удовлетворяет трём критериям корректности:
1. **Отсутствие пропусков дедлайнов** при загрузке процессора ≤ границы Liu & Layland.
2. **Корреляция пропусков** при превышении границы, но не более 100% загрузки.
3. **Верхняя граница времени работы планировщика** (предсказуемость) — не более 50 мкс на один тик в заданных условиях.

## Возможности программы

* Гибкое управление задачами (добавление, удаление, сброс).
* Верификация планируемости (расчёт утилизации и сравнение с границей Liu & Layland).
* Симуляция с произвольной длительностью (до 100000 тиков).
* Поддержка вытеснения с учётом времени переключения контекста.
* Сбор детальной статистики (переключения контекста, пропуски дедлайнов, время простоя, замеры времени работы планировщика) и экспорт в CSV.
* Сохранение и загрузка конфигураций задач для воспроизводимости результатов.
* Модульная архитектура: ядро планировщика (`scheduler/`) не зависит от операционной системы, все системные вызовы изолированы в модуле `utils/`. Это позволяет портировать программу на Linux с минимальными изменениями (замена WinAPI на POSIX-аналоги).


## Требования и установка

* Операционная система: Microsoft Windows 7 и выше.

Установка: 
1. Склонируйте репозиторий или скачайте архив.
2. Исполняемый файл `RTOS.exe` можно разместить в любом месте.
3. Запустите `RTOS.exe`.

Настройка:
* При первом запуске программа создаст в своей директории папку `logs` для хранения конфигураций, логов и результатов симуляции.
* Для работы с аутентификацией предусмотрены следующие тестовые учетные записи:
  - Администратор: admin/admin123
  - Пользователь: user/user123

## Режимы работы и роли

После запуска вы увидите консольное меню. В зависимости от выбранной роли (гость, пользователь, администратор) доступны различные функции:
* Гость: Только вход в систему.
* Пользователь: Добавление задач, запуск симуляции, просмотр статистики, сброс планировщика.
* Администратор: Полный доступ: управление пользователями, тестовые сценарии, генератор сценариев, юнит-тесты и управление конфигурациями задач.

## Работа с программой: 

Использование программы интуитивно понятно и строится вокруг трех основных этапов.

### 1. Загрузка и проверка системы:

* Начните с загрузки одного из тестовых сценариев или ручного добавления задач. Для каждого набора можно рассчитать общую загрузку ЦП (Utilization) и границу Liu & Layland (U_lub), чтобы оценить планируемость набора:
  - Если загрузка ≤ U_lub: набор гарантированно планируем.
  - Если U_lub < Загрузка ≤ 100%: планируемость возможна, но не гарантирована.
  - Если Загрузка > 100%: система непланируема.

### 2. Запуск симуляции:

* Запустите симуляцию на заданное количество тиков (от 1 до 100000). В режиме реального времени вы будете видеть:
  - Текущее системное время.
  - Какая задача выполняется (`RUNNING`) или когда процессор простаивает (`IDLE`).
  - Состояние задачи: `Начало`, `Выполняется`, `Завершение`, `Завершена`.
  - Список задач, которые активируются в данный момент.
  - Оставшееся время выполнения задачи.
* Наглядная демонстрация вытеснения задач более высоким приоритетом — одна из ключевых особенностей программы

### 3. Анализ результатов:

* После завершения симуляции программа выводит подробную статистику, которая также сохраняется в CSV-файл в папке logs/.
  - Статистика выполнения: Общее время симуляции, количество переключений контекста и пропущенных дедлайнов.
  - Анализ загрузки: Фактическая утилизация ЦП, сравнение с теоретической границей Liu & Layland и вывод о планируемости.
  - Время работы планировщика: Минимальное, максимальное и среднее время выполнения логики планирования (в микросекундах). Этот показатель доказывает предсказуемость работы планировщика и его независимость от общего количества задач.

## Тестирование:

Программа включает **6 встроенных юнит-тестов** для автоматической верификации ключевых свойств (доступны администратору):

* Проверка теоремы Liu & Layland.
* Граничный случай планируемости.
* Реакция на перегрузку (утилизация > 100%)..
* Корректность назначения приоритетов RMS.
* Обнаружение пропусков дедлайнов.
* Корректное добавление/удаление задач.

А также пять готовых сценариев нагрузки для интеграционного тестирования.

## Переносимость на другие ОС

Текущая версия ориентирована на Windows (используются WinAPI-вызовы: `SetConsoleCP`, `_mkdir`, `FindFirstFileA`, `_access`). Все системно-зависимые вызовы изолированы в модуле `utils/`, а для портирования на Linux достаточно переписать этот модуль, заменив WinAPI на POSIX-аналоги (`setlocale`, `mkdir`, `opendir/readdir`, `access`). Ядро планировщика (`scheduler/`) также остаётся неизменным, что подтверждает продуманную архитектуру.
