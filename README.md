# -Планировщик задач ОСРВ» (RTOS)-
Этот проект - симулятор планировщика реального времени, реализующий классический алгоритм Rate Monotonic Scheduling (RMS). Его цель - продемонстрировать детерминизм и предсказуемость алгоритма RMS, свойственные системам реального времени. Инструмент не просто моделирует работу планировщика, но и предоставляет все необходимое для его верификации и глубокого изучения.

Название: Планировщик задач ОСРВ» (RTOS)
* Язык: C++17 (STL, стандартные контейнеры, системный WinAPI для работы с кодировкой, файловой системой и таймерами)
* Назначение: Образовательный инструмент, который позволяет сгенерировать набор периодических задач и наглядно пронаблюдать за их выполнением планировщиком RMS. Включает систему аутентификации, логирование, экспорт данных, сохранение/загрузку конфигураций и набор юнит-тестов для проверки ключевых свойств алгоритма.

Алгоритм Rate Monotonic Scheduling (RMS) - это статический алгоритм планирования с фиксированными приоритетами, где каждая периодическая задача получает приоритет, обратно пропорциональный своему периоду: чем короче период, тем выше приоритет. Основой для анализа планируемости служит теорема Liu & Layland, согласно которой набор задач гарантированно планируем, если загрузка процессора не превышает определенный порог.

Программа позволяет:
* Гибкое управление задачами.
* Верификация планируемости (расчет утилизации и сравнение с границей).
* Симуляция с произвольной длительностью (до 100 000 тиков).
* Поддержка вытеснения с учетом времени переключения контекста.
* Сбор детальной статистики и экспорт.
* Сохранение и загрузка конфигураций для воспроизводимости результатов.

Для компиляции и запуска вам понадобятся:
* Операционная система: Microsoft Windows (7 и выше).

Установка (Get Started): 
* Склонируйте архив на свой компьютер.
* Возьмите файл RTOS.exe и положите на рабочий стол.
* Нажмите на программу RTOS.exe левой кнопкой компьютерной мыши.

Настройка:
* Перед первым запуском программа создаст в своей директории папку logs для хранения конфигураций, логов и результатов симуляции. Для работы с аутентификацией предусмотрены следующие тестовые учетные записи:
* Администратор: admin/admin123
* Пользователь: user/user123

После запуска вы увидите консольное меню. В зависимости от выбранной роли (гость, пользователь, администратор) доступны различные функции:
* Гость: Только вход в систему.
* Пользователь: Добавление задач, запуск симуляции, просмотр статистики, сброс планировщика.
* Администратор: Полный доступ: управление пользователями, тестовые сценарии, генератор сценариев, юнит-тесты и управление конфигурациями задач.

Работа с программой: Использование программы интуитивно понятно и строится вокруг трех основных этапов.

1. Загрузка и проверка системы:
* Начните с загрузки одного из тестовых сценариев или ручного добавления задач. Для каждого набора можно рассчитать общую загрузку ЦП (Utilization) и границу Liu & Layland (U_lub), чтобы оценить планируемость набора:
* Если загрузка ≤ U_lub: набор гарантированно планируем.
* Если U_lub < Загрузка ≤ 100%: планируемость возможна, но не гарантирована.
* Если Загрузка > 100%: система непланируема.

2. Запуск симуляции:
* Запустите симуляцию на заданное количество тиков (от 1 до 100 000). В режиме реального времени вы будете видеть:
* Текущее системное время.
* Какая задача выполняется (RUNNING) или когда процессор простаивает (IDLE).
* Её состояние: Начало, Выполняется или Завершение/Завершена.
* Список задач, которые активируются в данный момент.
* Оставшееся время выполнения задачи.
* Наглядная демонстрация вытеснения задач более высоким приоритетом — одна из ключевых особенностей программы

3. Анализ результатов:
* После завершения симуляции программа выводит подробную статистику, которая также сохраняется в CSV-файл в папке logs/.
* Статистика выполнения: Общее время симуляции, количество переключений контекста и пропущенных дедлайнов.
* Анализ загрузки: Фактическая утилизация ЦП, сравнение с теоретической границей Liu & Layland и вывод о планируемости.
* Время работы планировщика: Минимальное, максимальное и среднее время выполнения логики планирования (в микросекундах). Этот показатель доказывает предсказуемость работы планировщика и его независимость от общего количества задач.

Структура репозитория:
* Форматы файлов в в папке logs/:
* Логи: Текстовые файлы (.log).
* Конфигурации: Текстовые файлы (.txt) с форматом: Имя;Период;WCET;Дедлайн.
* Результаты симуляции: Файлы (.csv) с разделителем-точкой с запятой, содержащие сводную статистику и таблицу задач.

Тестирование:
* Программа включает в себя 6 встроенных юнит-тестов для автоматической верификации ключевых свойств:
* Проверка теоремы Liu & Layland.
* Проверка граничного случая планируемости.
* Проверка реакции на перегрузку (утилизация > 100%).
* Проверка корректности назначения приоритетов.
* Проверка обнаружения пропусков дедлайнов.
* Проверка корректного добавления/удаления задач.
* А также пять готовых сценариев нагрузки для интеграционного тестирования.
